Tavern in the Square: Case Study


Tavern in the Square is a bar and restaurant with locations throughout Boston and Cambridge. Each location features different drinks and food. Because of this varying menu, the Tavern in the Square website faces navigation issues.

The Challenge


Evaluate the usability of the Tavern in the Square website, especially the over-arching navigation bar and the "Menu" navigation item. The current website has various navigation issues due to the varying menus. These issues were investigated through three usability studies.

Menu Page

The Process


Three usability tests were conducted to test the navigation bar and the "Menu" navigation item. Example tasks included "find the restaurant closest to you", "choose a craft beer on tap at the Allston location", and "find the hours of the Central Square location on Saturday".

Using the notes gathered from the three usability studies, the following insights were drawn out:

  1. Navigation bar has too many options and is spread out across the top and the bottom of the main page.
  2. Menu pages feature many broken images, are very text-heavy, and show no prices.
  3. Without prior knowledge of Boston, it's hard to determine where the closest location was.
  4. Users expected to find the "drink" menu on the "Menu" navigation item, rather than on the main page.

From these notes, three options were sketched to redesign the menu page to accommodate the difference in menus across locations.

The Execution


These options were then combined into one finalized option and mocked up in Balsamiq. The finalized option allowed the browser to auto-detect and display the nearest location when a user clicked on the Menu option in the navigation bar. If the user was not interested in this particular location, they could click on the "Change Location" button underneath to be brought to the Locations page.

Design Rationale

  1. Navigation Bar: The top navigation bar was cleaned up to have four main options - Locations, Menus, Our Story, and Contact. The social media links were dislayed underneath the main logo.
  2. Secondary Navigation: Secondary navigation items (such as the "Apply" and "Gift Card") were moved to the bottom footer.
  3. Menu Page: The menu page added in pictures and prices and broke the menu into five categories. This allows for the user to click into the menu they are interested in and breaks up the text.
  4. Closest Location: Auto-detect location of browser and display the closest location.
  5. Drink Category: Drink menu is moved from the front page to the individual location pages. The drink menu is moved to the top of the menu offerings, as Tavern in the Square is known for its bar offerings and its draft selection.
